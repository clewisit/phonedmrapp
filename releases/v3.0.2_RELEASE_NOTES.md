## 📚 Feature Guide: How to Use Everything

v3.0.2 includes ALL features from v1.0-v1.7.0. Here's how to use them:

### 🎙️ Voice Transcription

Convert received voice transmissions to text using OpenAI Whisper API.

**Quick Start**:
1. Get API key from https://platform.openai.com/api-keys
2. File will be at `/storage/emulated/0/DMR/api_key.txt`(created automatically in v1.7.0)
3. Paste your OpenAI API key into the file
4. Restart DMR app
5. Toggle "TXT" button (green = enabled)

**What You Get**:
- Real-time transcription displayed in borderbox
- Per-channel history (last 3 transmissions)
- Timestamps and caller identification
- Cost: ~$0.006 per minute of audio

**Button**: Green "TXT" below PTT on left side

### 🎤 Audio Recording

Record all incoming transmissions to WAV files.

**Quick Start**:
1. Toggle "REC" button (red = enabled)
2. Recording auto-starts when receiving
3. Files saved to `/storage/emulated/0/DMR/Recordings/[ChannelName]/`

**File Details**:
- Format: WAV (16kHz, 16-bit, mono)
- Names: `YYYYMMDD_HHmmss_ContactName.wav`
- Per-channel folders for organization
- Empty files (<10KB) auto-deleted

**Button**: Red pill-shaped "REC" below PTT on right side

### 📍 Location Display

Shows where each channel's location is (city, state, elevation).

**What You See**:
- **Top-right corner**: `Minneapolis, Minnesota\n859ft (262m) 📍`
- **Updates**: Automatically when switching channels
- **Fallback**: Shows 📍 icon if no location data

**Adding Locations**:
1. Export channels (Backup/Restore → Export)
2. Edit `Channels.csv` - add `Latitude` and `Longitude` columns
3. Example: `44.9203`, `-93.2654` (decimal degrees)
4. Import CSV back
5. Location appears automatically

**Services Used**:
- Android Geocoder (city/state lookup)
- Open-Elevation API (elevation, free)

### 📊 Signal Strength (RSSI)

Real-time signal strength indicator.

**Display**:
- **Yellow box** above caller info
- **Format**: `Signal: -85 dBm`
- **Saves**: Values stored in activity history

**Reading RSSI**:
- `-50 to -70 dBm`: Excellent (full bars)
- `-70 to -85 dBm`: Good (3-4 bars)
- `-85 to -95 dBm`: Marginal (1-2 bars)
- `-95 to -110 dBm`: Poor (barely usable)

### 📜 DMR Activity History

Per-channel log of last 3 transmissions/events.

**Display**:
- **Blue box** in top-right corner
- **Shows**: Contact name, time, activity type, signal strength
- **Example**: `JohnDoe 09:15:23 Voice RX -82 dBm`

**Activity Types**:
- Voice RX/TX
- SMS messages
- Channel busy
- Low battery warnings

**Persistence**: Survives channel switches and app restarts

### 🏷️ Caller Identification

Shows who's talking during DMR transmissions.

**Display**:
- **Top-left** of borderbox (green text)
- **With contact**: `📞 Contact Name\nDMR ID: 64067`
- **Without contact**: `📞 DMR ID: 64067`
- **Auto-clears**: When transmission ends

**Setup**: Add contacts via app's Contacts menu for name display

### 📦 CSV Backup/Restore

Full backup compatible with OpenGD77 CPS software.

**Export**:
1. LOCAL tab → "Backup/Restore"
2. "Export to OpenGD77 CSV"
3. Files in `/storage/emulated/0/Download/DMR/DMR_Backups/[timestamp]/`

**Files Created**:
- `Channels.csv` (28 parameters)
- `Contacts.csv` (DMR contacts)
- `TG_Lists.csv` (Talk groups)
- `Zones.csv` (Zone assignments)
- `DTMF.csv` (DTMF contacts)
- `Backup_Summary.pdf` (Human-readable)
- `DMR_Backup_[timestamp].zip` (All files)

**Import**:
1. LOCAL tab → "Backup/Restore"
2. "Import from OpenGD77 CSV"
3. Select backup folder
4. Channels imported and refreshed automatically

**PC Workflow** (Optional):
1. Export from phone
2. Transfer to PC via USB
3. Edit in OpenGD77 CPS software
4. Export back to CSV
5. Transfer to phone
6. Import on phone

**Supported**:
- ✅ Digital & Analog channels
- ✅ CTCSS/DCS tones
- ✅ Power levels (auto-conversion)
- ✅ Squelch settings
- ✅ Contact names
- ✅ Locations (lat/long)

## ⚙️ Tips & Tricks

### Battery Optimization
- Disable transcription when not needed (TXT button off)
- Disable recording when not needed (REC button off)
- Both save battery and storage

### Storage Management
- Recordings in Download/DMR folder (easy USB access)
- Delete old recordings via file manager
- Each channel gets its own subfolder

### Best Practices
1. **Always export before major changes** (backup safety)
2. **Test imports with one channel first** (verify compatibility)
3. **Name channels clearly** (easier to find recordings)
4. **Keep API key secure** (costs money if exposed)
5. **Monitor RSSI** (relocate if signal poor)

### Transcription Accuracy
- Works best with clear audio
- English language optimized
- Background noise reduces accuracy
- Short transmissions may not transcribe

### OpenGD77 CPS Compatibility
- Supports all OpenGD77 CSV formats
- Power levels auto-convert (P1-P9)
- Squelch percentages auto-convert
- CTCSS/DCS tones preserved
- Extra columns (lat/long) ignored by OpenGD77

## 💡 Troubleshooting

### Transcription Not Working
- ✅ Check `/storage/emulated/0/DMR/api_key.txt` exists
- ✅ Verify API key is valid (test on OpenAI website)
- ✅ Enable REC button (transcription needs recording)
- ✅ Check internet connection
- ✅ View logs: `adb logcat | grep DMRModHooks`

### No Location Showing
- ✅ Export channels and check for lat/long columns
- ✅ Import CSV with location data
- ✅ Switch to different channel and back
- ✅ Shows 📍 if no data (expected)

### RSSI Not Displaying
- ✅ Must be during active reception
- ✅ Space reserved when idle (shows "Signal: " text)
- ✅ Both DMR and analog supported
- ✅ Check logs for SignalMessageHandler errors

### Recordings Empty/Missing
- ✅ Enable REC button before/during reception
- ✅ Files <10KB auto-deleted (too short)
- ✅ Check folder: `Download/DMR/Recordings/[ChannelName]/`
- ✅ Verify storage permissions granted

### MON Button Not Visible
- ✅ Only shows on analog channels
- ✅ Hidden on digital (DMR work in progress)
- ✅ Switch to analog FM channel
- ✅ See v2.x for DMR monitoring (future)

### Import Fails
- ✅ Verify CSV format matches OpenGD77
- ✅ Check Channels.csv has required columns
- ✅ Test with single channel first
- ✅ View logcat for specific errors
